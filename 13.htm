Unban a player

In this article, we'll show you how to easily unban a player using our system on your Aternos Minecraft server.

Video tutorial: 

You can unban a player here: https://aternos.org/players/banned-players

Or you can unban the player with the command /pardon [name] in the console when your server is online. Open the console here: https://aternos.org/console/

 

If you've banned the player IP you can unban the player here: https://aternos.org/players/banned-ips

Or you can unban the player with the command "/pardon-ip [ip]" in the console when your server is online. To get the IP address, list all IP bans with the command "/banlist ips".

Change your Gamemode

To change your gamemode on a minecraft server use the /gamemode command.

This is how it is used: 

/gamemode <mode> [player]

Example:

/gamemode creative Notch

The available gamemodes are: survival, creative, adventure, spectator. What the different gamemodes are for and what they do exactly is explained here.

The name of the player is only required if you want to change the gamemode for another player or if you use that command from the console.

You have to be OP!

Please note that you have to be OP in order to use that command. This tutorial shows on how to get OP: https://www.youtube.com/watch?v=QfjVQSp9enc

If you use a permission system make sure that you have the appropriate permissions assigned to your group/player

Delete someones player data

A user's player data contains any player-related data on a server, like their inventory, achievements, location, statistics, ender chest etc.

Sometimes it's required to delete someone's player file. For example, if they have a broken item in their inventory.

Deleting someone's player file causes any data related to the player to be reset. This is sadly only possible on Java Edition, as Bedrock Edition stores player data and world files together.

 

Delete a player file

Before deleting the player data, make sure that your server is offline. Otherwise, the server might overwrite the deleted data again when it saves.
